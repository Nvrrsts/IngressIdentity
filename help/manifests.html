Manifest File Format

A manifest file is a spreadsheet containing the following columns:

lastupdated

Required.A unique number. This number has to be updated every time the source sheet this row describes is updated.

Proposed value:

YYYYMMDDXX
where YYYY is the year, MM is the month, DD is the day and XX is the lowest positive number that results in a unique number, e.g. 2014043001.

tag

Required.A human readable name for the source. The extension will display this instead of the key.

faction

Required1.Either 'enlightened', 'resistance' or 'unknown'.

key

Required.The key of the source file. This tells the extension where to find the spreadsheet. For more information, see this help section on keys.

refresh

Required.The amount of hours to wait before updating this source file again. The extension may take up to one hour longer than the alotted refresh rate.

All integer values are allowed. Negative values are the same as 0: reload every hour.

…

Optional.You can create other columns at will. These will be optional, and will be seen as extratags. If the column name is also added in extratags, the data in extratags is overwritten.

anomaly

One of '13magnus', 'recursion', 'interitus' or 'helios'. Other values are ignored. New values will be added for future anomalies.

community
event

A value of type

OID: name

For more information on the OID, see the tools page.

The name can be anything as long as the cell contains valid JSON.

Example: the spreadsheet has an 'anomaly' column. The data in this column will overwrite the 'anomaly' value in the manifest if that value exists.

Note 1: The faction has to be defined at least once for every player. You can omit this data if you are sure that all players in the source file are already defined elsewhere. This allows you to e.g. have a source for an event where you include both factions at the same time. See the compatibility page for more information.
